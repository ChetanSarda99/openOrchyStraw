## Overview

This example shows a minimal OrchyStraw setup with 3 agents: a backend developer, a frontend developer, and a PM coordinator. Good for small-to-medium web projects.

## agents.conf

```ini
# id | prompt_path | ownership | interval | label
02-backend  | prompts/02-backend/02-backend.txt  | src/api/ src/models/ src/middleware/     | 1 | Backend Developer
03-frontend | prompts/03-frontend/03-frontend.txt | src/components/ src/pages/ src/styles/  | 1 | Frontend Developer
01-pm       | prompts/01-pm/01-pm.txt             | prompts/                                | 0 | PM Coordinator
```

---

Building a task management app with Express + PostgreSQL.
Tech stack: TypeScript, Express, Prisma ORM, PostgreSQL.

## What Exists — 12 Files
8 TypeScript source + 4 test files. All tests passing.

## YOUR TASKS

## Running it

```bash
# First cycle — both agents build their initial work
./scripts/auto-agent.sh

# Three more cycles — PM coordinates, agents iterate
./scripts/auto-agent.sh --cycles 3
```

After the first cycle, shared context might look like:

```markdown
## Backend Status
- Created CRUD endpoints for /api/tasks (GET, POST, PUT, DELETE)
- Response format: { data: Task[], total: number, page: number }
- Zod validation on POST/PUT bodies

## Frontend Status
- Created TaskList component with pagination
- Created TaskForm component (create mode)
- NEED: API response type definitions from backend
```

The PM reads this, updates prompts for cycle 2, and the agents continue building.
