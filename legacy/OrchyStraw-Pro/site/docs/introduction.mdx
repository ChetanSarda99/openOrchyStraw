# What is OrchyStraw?

OrchyStraw runs a team of AI coding agents on any codebase. You define agents in a config file, write their prompts in markdown, and a single bash script orchestrates everything — spawning agents in parallel, managing file ownership, coordinating via shared context, and merging cleanly.

No Python runtime. No framework. No dependencies. Just bash and your AI tool of choice.

## How it works

<Steps>
  <Step title="Configure your agents">
    Define roles in `agents.conf` — backend, frontend, QA, PM, whatever your project needs. Each agent gets a prompt file and file ownership boundaries.
  </Step>
  <Step title="Run the orchestrator">
    One command: `./auto-agent.sh`. Agents read their prompts, do their work in parallel, and write status to shared context. No conflicts — file ownership prevents agents from stepping on each other.
  </Step>
  <Step title="Review and ship">
    The PM agent coordinates after workers finish. QA reviews. The script commits by file ownership, merges to main, and starts the next cycle. You review and merge when ready.
  </Step>
</Steps>

## Architecture

OrchyStraw uses a **parallel workers + sequential PM** pattern:

1. **Workers run simultaneously** — backend, frontend, QA all work at the same time (3x faster than sequential)
2. **PM runs last** — single coordination point that reviews work, updates tasks, and plans the next cycle
3. **Script controls git** — agents never touch branches, commits, or merges

This is intentionally a hub-and-spoke topology, not a mesh. The script is the hub. Agents are spokes.
