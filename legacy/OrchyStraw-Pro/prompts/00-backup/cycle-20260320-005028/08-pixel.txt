## What is Pixel Agents?

Pixel Agents (by Pablo de Lucca) is a pixel art office visualization that maps
AI coding agent activity to animated characters. Originally built for Claude Code,
we're adapting it for OrchyStraw's multi-agent orchestration.

**Original:** https://github.com/pablodelucca/pixel-agents (MIT)
**Standalone fork:** https://github.com/rolandal/pixel-agents-standalone (MIT)
**Research:** `research/PIXEL-AGENTS-INTEGRATION.md`

**Repo:** https://github.com/ChetanSarda99/OrchyStraw

---

## The Vision

Every OrchyStraw cycle, agents appear in the pixel art office:
- **Backend agent** sits at their desk and starts typing
- **Frontend agent** walks in, sits down, starts coding
- **QA agent** reads files when their interval hits
- **PM agent** walks between desks, updating prompts after workers finish
- **Idle agents** sit still or wander — showing they haven't run this cycle
- **Shared context** shows as agents "talking" (speech bubble)

## Architecture

### Approach 1: Synthetic JSONL Emitter (SHIP FIRST)
Auto-agent.sh writes JSONL events in Pixel Agents format during cycles.

### Approach 3: Fork + OrchyStraw Adapter (LONG-TERM)

## JSONL Event Mapping

| OrchyStraw Event | JSONL Type | Character Animation |
|-----------------|-----------|-------------------|
| Agent spawned | tool_use: write_file | Walk to desk, start typing |
| Agent reading prompt | tool_use: read_file | Reading animation |
| Agent running command | tool_use: bash | Running animation |
| Agent writing to shared context | tool_use: write_file + speech | Speech bubble |
| Agent idle (not this cycle) | (no events) | Sitting idle / wandering |
| Agent finished | turn_end | Walk to idle spot |
| Agent error/crash | error event | Red flash, slump |
| PM updating prompts | tool_use: write_file (multi) | Walking between desks |
