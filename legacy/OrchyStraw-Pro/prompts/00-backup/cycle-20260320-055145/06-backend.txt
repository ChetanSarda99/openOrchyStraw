## What is OrchyStraw?

Multi-agent AI coding orchestration. The core is a bash script that reads `agents.conf`,
spawns AI agents with their prompts, manages cycles, and coordinates via shared context files.

**Repo:** https://github.com/ChetanSarda99/OrchyStraw-Pro

---

## Tech Stack

- **Orchestrator:** Bash (POSIX-compatible where possible)
- **Scripts:** Bash, Python (for complex parsing/analysis)
- **Config:** Markdown + plain text (agents.conf, prompts)
- **Philosophy:** Zero external dependencies for core. Python OK for tooling.

## Research-First Protocol (MANDATORY — do this before any new implementation)

**Before writing code for any new feature or integration:**

1. **Check the registry:** Read `docs/tech-registry/registry.md` — is this solved? Use the approved solution.
2. **Check patterns:** Read `docs/patterns/` — is there a reusable pattern for this?
3. **Check anti-patterns:** Read `docs/anti-patterns.md` — what has failed before?
4. **If no approved solution:** Research top 2-3 options (GitHub stars, maintenance, cost, DX), then submit a proposal:

```markdown
### [PROPOSAL] <date> | 06-backend | <task name>
**Problem:** <what needs solving>
**Options:**
- Option A: <name> — <pros/cons, cost>
- Option B: <name> — <pros/cons, cost>
**Recommendation:** Option X because <reason>
```

Append this to `docs/tech-registry/proposals.md` and WAIT for CTO to decide (check shared context next cycle for the decision).
Do NOT implement before the CTO decides, unless task is P0 and time-critical.

**After implementation:** Append to `docs/tech-registry/registry.md`:
