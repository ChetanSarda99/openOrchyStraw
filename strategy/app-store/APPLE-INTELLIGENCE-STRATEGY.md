# Apple Intelligence Strategy — Rekall

**Decision Date:** March 16, 2026
**Status:** Implementation complete (code written, needs Mac for runtime testing)

---

## Executive Summary

Rekall uses a **hybrid AI architecture**: Apple Intelligence handles high-volume, simple tasks on-device (free, private, offline), while server-side models handle complex tasks requiring large context windows or cross-note reasoning.

**Estimated cost savings:** 40-60% reduction in Claude/AssemblyAI API costs.

---

## What Moves On-Device

### 1. Categorization / Tagging — Foundation Models + @Generable

| | Before | After |
|---|--------|-------|
| **Service** | Claude 3.5 Sonnet (server) | Foundation Models (on-device) |
| **Cost** | ~$0.003-0.015/note | Free |
| **Latency** | 500-2000ms (network + inference) | 100-300ms (on-device) |
| **Offline** | No | Yes |
| **Quality** | Excellent | Good — `@Generable` guarantees valid structured output |

**How it works:** The `@Generable` macro with constrained decoding forces the model to output valid Swift types matching our categorization schema. The model *cannot* output invalid categories — it's forced to pick from our enums (8 life areas, 8 formats, 4 statuses).

```swift
@Generable
struct NoteClassification {
    @Guide(description: "The content format type")
    let format: NoteFormat
    @Guide(description: "The primary life area")
    let area: LifeArea
    @Guide(description: "3-5 relevant tags")
    let tags: [String]
    @Guide(description: "Status: inbox, review, reference, archive")
    @Guide(.anyOf(["inbox", "review", "reference", "archive"]))
    let status: String
}
```

**Requirements:** iOS 26+, iPhone 15 Pro+ (A17 Pro), Apple Intelligence enabled, ~7GB storage
**Fallback:** Backend `POST /notes/:id/categorize` via Claude

### 2. Short Note Summarization — Foundation Models

| | Before | After |
|---|--------|-------|
| **Service** | Claude 3.5 Sonnet (server) | Foundation Models (on-device) |
| **Cost** | ~$0.003-0.01/note | Free |
| **Limit** | 200K tokens | **4,096 tokens total** (input + output) |
| **Offline** | No | Yes |

**Constraint:** The 4,096 token context window limits this to notes under ~3,000 characters. That covers tweets, Telegram messages, Instagram captions, voice memo transcripts, and most bookmarked links. Longer content (articles, Notion pages) still goes to Claude.

**Requirements:** iOS 26+, iPhone 15 Pro+
**Fallback:** Backend `POST /notes/:id/summarize` via Claude (always used for long content)

### 3. Action Extraction — Foundation Models + @Generable

| | Before | After |
|---|--------|-------|
| **Service** | Claude 3.5 Sonnet (server) | Foundation Models (on-device) |
| **Cost** | ~$0.003/note | Free |
| **Quality** | Excellent (nuanced) | Good for simple actions |

**How it works:** Same `@Generable` pattern — model extracts action items with priority and urgency flags, constrained to valid output types.

**Limitation:** Per-note extraction only. Cross-note pattern analysis (e.g., "you've been saving a lot about sleep") still requires Claude's large context window.

**Requirements:** iOS 26+, iPhone 15 Pro+
**Fallback:** Backend extraction via Claude

### 4. Voice Transcription — SpeechAnalyzer

| | Before | After |
|---|--------|-------|
| **Service** | AssemblyAI ($0.00025/sec) | SpeechAnalyzer (on-device) |
| **Cost** | ~$0.015/min | Free |
| **WER** | ~5-8% (cloud) | ~14% (on-device) |
| **Languages** | 100+ | 10 |
| **Offline** | No | Yes |
| **Diarization** | Yes | No |

**Trade-off:** Higher word error rate (14% vs 5-8%), but free and works offline. For personal voice memos ("remind me to buy groceries", "idea for the blog post"), 14% WER is perfectly acceptable. The privacy benefit (audio never leaves the device) is significant for ADHD users.

**Requirements:** iOS 26+, must check `SpeechTranscriber.supportsDevice()`
**Fallback:** Backend AssemblyAI (for unsupported languages or when user prefers accuracy)

### 5. Offline Semantic Search — NLContextualEmbedding

| | Before | After |
|---|--------|-------|
| **Service** | Voyage AI voyage-3 (1024-dim, server) | NLContextualEmbedding (512-dim, on-device) |
| **Use case** | Primary search | Offline search only |
| **Quality** | Best-in-class | Good enough for local search |
| **Offline** | No | Yes |

**This is NOT a replacement for Voyage.** NLContextualEmbedding provides 512-dim vectors on iOS (half the capacity of Voyage's 1024-dim). It's used for offline/instant local search against cached notes. The canonical embeddings stored in pgvector are still generated by Voyage server-side.

**Requirements:** iOS 17+ (all devices — no Apple Intelligence requirement)
**Fallback:** N/A — this is an additive capability, not a replacement

## What Stays Server-Side

| Capability | Why It Can't Move On-Device |
|------------|---------------------------|
| **RAG Chat ("Chat with your notes")** | 4K token limit = can fit ~1-2 notes as context. Real RAG needs 10-50 notes. Claude's 200K context is required. |
| **Long-form summarization** | Articles, Notion pages, threads exceed 4K tokens. |
| **Cross-note pattern detection** | "You saved 7 things about sleep this month" requires processing many notes simultaneously. |
| **Graph clusters** | Needs to analyze all user notes' embeddings — too many for on-device. |
| **Weekly digest generation** | Synthesizes a week's worth of notes — large context required. |
| **Canonical embeddings** | Voyage-3 quality is materially better for the primary search index. |
| **High-accuracy transcription** | AssemblyAI for long recordings, multi-speaker, or 90+ languages. |

## User-Facing Settings

In Settings → On-Device AI:
- **Toggle:** "Process notes on device when possible" (default: ON)
- **Status:** "Apple Intelligence: Available / Not available on this device"
- **Info:** "Uses Apple Intelligence for categorization, summarization, and voice transcription. Reduces data sent to servers."

When Apple Intelligence is unavailable (older devices, not enabled), the app works identically — just routes everything through the backend API. No feature loss, only cost/privacy/offline trade-offs.

## Device Compatibility

| Feature | Minimum iOS | Device Requirement |
|---------|------------|-------------------|
| Core app (all features) | iOS 17+ | Any iPhone with iOS 17 |
| Offline search (NLContextualEmbedding) | iOS 17+ | Any device |
| OCR (Apple Vision) | iOS 17+ | Any device |
| On-device categorization | iOS 26+ | iPhone 15 Pro+ (A17 Pro) |
| On-device summarization | iOS 26+ | iPhone 15 Pro+ (A17 Pro) |
| On-device action extraction | iOS 26+ | iPhone 15 Pro+ (A17 Pro) |
| On-device transcription | iOS 26+ | Must pass `SpeechTranscriber.supportsDevice()` |

**Decision:** iOS 17 remains the minimum. Apple Intelligence is an optional enhancement, not a requirement. This keeps the addressable market wide while giving newer device owners a premium experience.

## Future: Custom Adapter Training

Apple supports LoRA adapter training for the on-device Foundation Model. We could train a custom adapter specifically for Rekall's 5-dimension categorization system, potentially matching Claude's accuracy for that specific task.

- **Training:** Python toolkit on Mac/Linux with Apple silicon
- **Deployment:** Via Background Assets framework (~160MB per adapter)
- **Caveat:** Must retrain with each base model update
- **Priority:** Post-launch optimization. The generic Foundation Model with `@Generable` constraints is good enough for MVP.
