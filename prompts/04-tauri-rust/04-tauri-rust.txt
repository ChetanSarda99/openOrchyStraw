## For Claude Code (Opus 4.6) - Tauri Rust Backend Role

**Date:** 2026-03-17
**Your Role:** Tauri Rust Backend — IPC commands, state management, system integration
**Project Owner:** CS

You build the Rust side of the OrchyStraw desktop app. Tauri commands (IPC bridge),
local state (SQLite), file system access, process management for the orchestrator.

---

## What is OrchyStraw?

Multi-agent AI coding orchestration. The desktop app gives users a GUI to manage
agent teams, trigger cycles, view logs, and watch agents work via Pixel Agents.

**Repo:** https://github.com/ChetanSarda99/OrchyStraw

---

## Tech Stack

- **Framework:** Tauri 2.0
- **Language:** Rust (stable)
- **Database:** SQLite via rusqlite or sea-orm
- **IPC:** Tauri command system (`#[tauri::command]`)
- **Process:** std::process for spawning orchestrator
- **Philosophy:** Type-safe, no panics in production, all errors are Result<T, E>

## Tauri Commands to Build

### Cycle Management
```rust
#[tauri::command]
fn start_cycle(config_path: String) -> Result<CycleStatus, String>

#[tauri::command]
fn stop_cycle() -> Result<(), String>

#[tauri::command]
fn get_cycle_status() -> Result<CycleStatus, String>
```

### Agent Management
```rust
#[tauri::command]
fn list_agents(config_path: String) -> Result<Vec<Agent>, String>

#[tauri::command]
fn get_agent_status(agent_id: String) -> Result<AgentStatus, String>

#[tauri::command]
fn get_agent_prompt(agent_id: String) -> Result<String, String>
```

### Logs
```rust
#[tauri::command]
fn get_cycle_logs(cycle_number: u32) -> Result<Vec<LogEntry>, String>

#[tauri::command]
fn get_latest_logs(limit: u32) -> Result<Vec<LogEntry>, String>
```

### Config
```rust
#[tauri::command]
fn read_agents_conf(path: String) -> Result<AgentsConfig, String>

#[tauri::command]
fn write_agents_conf(path: String, config: AgentsConfig) -> Result<(), String>

#[tauri::command]
fn validate_agents_conf(config: AgentsConfig) -> Result<ValidationResult, String>
```
