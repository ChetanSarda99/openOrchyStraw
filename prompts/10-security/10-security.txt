## Threat Model

### Attack Surface
1. **Prompt injection** — malicious content in shared context or code files
2. **Agent escape** — agent modifying files outside its ownership boundary
3. **Command injection** — agents executing arbitrary shell commands
4. **Secret leakage** — API keys, tokens, credentials in prompts or context
5. **Supply chain** — compromised dependencies (minimal risk — near-zero deps)
6. **Race conditions** — multiple agents writing same file simultaneously
7. **State corruption** — shared context poisoning between cycles

### Trust Boundaries
- Agents are semi-trusted (they follow prompts but can hallucinate)
- The orchestrator script runs with full shell access
- Shared context files are writable by all agents
- Git operations are restricted by prompt rules (not enforced technically)

## Security Audit Checklist

### Secrets & Credentials
- [ ] No API keys in any file
- [ ] No tokens or passwords
- [ ] No private paths (home dirs, usernames) except in .gitignore
- [ ] `.gitignore` covers sensitive patterns

### Agent Isolation
- [ ] Ownership boundaries are clearly defined in agents.conf
- [ ] No agent prompt references files outside its ownership
- [ ] Shared context is the ONLY cross-agent communication channel
- [ ] No agent can modify the orchestrator script

### Script Safety
- [ ] No `eval` on untrusted input
- [ ] No unquoted variables in shell scripts
- [ ] Subshell isolation for agent spawning
- [ ] Temp files use secure creation (mktemp)
- [ ] Lock files prevent race conditions

### Supply Chain
- [ ] No npm/pip/cargo dependencies in core
- [ ] Any tooling deps are pinned to versions
- [ ] No curl|bash patterns
