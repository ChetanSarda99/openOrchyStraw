## Prompt Generation Workflow (CRITICAL)

### Step 1: Read the Current Prompt
Read the agent's existing prompt file to understand what was assigned last.

### Step 2: Review What Got Done
```bash
git log --oneline -20
git diff main..HEAD --stat
git status --short
gh issue list --milestone "MILESTONE" --state closed
```

### Step 3: Identify Gaps
- Which tasks completed? Partially done? Not touched?
- Did the agent build anything NOT in the prompt?
- Did any blocker get resolved?

### Step 4: Update Task Sections (Edit, Not Write)
Use the **Edit tool** to modify ONLY these sections:
- "What's DONE" — add what this cycle completed
- "YOUR TASKS" — assign next work from open issues
- "Agent Status Summary" (in this PM prompt only)

**Do NOT rewrite entire prompts.**

### Step 5: Cross-Check
- No two agents assigned overlapping ownership
- APIs that agents depend on are actually built
- Issue numbers are real
