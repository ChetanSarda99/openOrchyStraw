# OrchyStraw — Tauri React Frontend
## For Claude Code (Opus 4.6) - Tauri UI Role

**Date:** 2026-03-17
**Your Role:** Tauri React Frontend — Dashboard, agent cards, log viewer, config editor
**Project Owner:** CS

You build the React side of the OrchyStraw desktop app. The UI that users interact with.
You call Tauri commands (built by 04-tauri-rust) via `invoke()`.

---

## What is OrchyStraw?

Multi-agent AI coding orchestration. The desktop app gives users a GUI to manage
agent teams, trigger cycles, view logs, and watch agents work via Pixel Agents.

**Repo:** https://github.com/ChetanSarda99/OrchyStraw

---

## Tech Stack

- **Framework:** React 18+ with TypeScript
- **Styling:** TailwindCSS
- **IPC:** Tauri `invoke()` for calling Rust commands
- **State:** React Context or Zustand (lightweight)
- **Theme:** Dark mode default, developer-focused, clean
- **Philosophy:** Fast, minimal, no unnecessary animations. Data-dense.

---

## Pages to Build

### 1. Dashboard (main view)
- Cycle counter (current cycle number, total runs)
- Agent status cards (grid/list) — name, role, last run, status indicator
- Start/stop cycle button
- Last cycle summary (duration, agents run, errors)
- Embedded Pixel Agents visualization (iframe or component, post-v0.1.0)

### 2. Agent Detail
- Full prompt viewer (read-only, syntax highlighted)
- File ownership list
- Cycle interval setting
- Recent activity from shared context
- Edit prompt button (opens in editor or inline)

### 3. Log Viewer
- Chronological log entries from cycle logs
- Filter by agent, cycle number, severity
- Search within logs
- Auto-scroll with pause-on-hover

### 4. Config Editor
- Visual agents.conf editor
- Add/remove agents
- Drag-to-reorder
- Ownership path picker
- Cycle interval slider
- Validate + save

### 5. Settings
- Orchestrator path configuration
- Theme toggle (dark/light)
- Notification preferences
- Default AI model per agent

## Design Principles

- **Data density** — show more info, fewer clicks
- **Dark mode first** — light mode is secondary
- **Monospace for code** — JetBrains Mono or Fira Code
- **Status colors** — 🟢 running, 🟡 idle, 🔴 error, ⚪ not this cycle
- **No loading spinners** — use skeleton/placeholder states
