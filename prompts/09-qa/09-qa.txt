## Anti-Pattern Duty (MANDATORY)

When you find a recurring bug or a pattern that failed:
1. Check `docs/anti-patterns.md` — is it already documented?
2. If not, append:
```markdown
## AP-XXX: <short title>
**Discovered:** <date> by 09-qa
**Problem:** <what went wrong>
**What was tried:** <what the agent did>
**Fix:** <correct approach>
**Affects:** <which parts of the project>
```

When a pattern WORKS well (tested, clean, reusable):
1. Append to `docs/patterns/<domain>/<name>.md`
2. Flag in shared context: `[QA PATTERN] Added <name> to patterns library`

1. Read `prompts/00-shared-context/context.md`

## QA Checklist (Run Every Session)

### Orchestrator Script
- [ ] `agents.conf` parsing handles edge cases (blank lines, comments, spaces)
- [ ] Cycle count increments correctly
- [ ] Agent spawn order matches config
- [ ] PM runs last (cycle_interval=0)
- [ ] Shared context is reset between cycles
- [ ] Backup of previous context is saved
- [ ] Error in one agent doesn't crash entire cycle
- [ ] SIGINT cleanup works (no orphan processes)

### Documentation
- [ ] README examples actually work
- [ ] All linked files exist
- [ ] No stale references to old names/paths
- [ ] agents.conf format matches what the script expects
- [ ] Template files are complete and usable

### Tauri App
- [ ] `cargo check` passes in src-tauri/
- [ ] React build succeeds
- [ ] Tauri commands return correct types
- [ ] IPC bridge matches between Rust commands and React invoke() calls
- [ ] Dark mode renders correctly

### Pixel Agents
- [ ] JSONL events match Pixel Agents format exactly
- [ ] Characters map to correct agents
- [ ] Animations trigger on correct events
- [ ] No stale JSONL files between cycles

### Prompts
- [ ] No ownership overlaps between agents (10 agents now)
- [ ] All agents reference correct repo URL
- [ ] Git safety rules present in every prompt
- [ ] No hardcoded paths that won't work on other machines
