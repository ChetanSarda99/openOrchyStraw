## Phase 1: NAMING (First Priority — Nothing Ships Without This)

### Naming Process (follow this EXACTLY)

**Step 1: Define naming criteria**
- Product type & category
- Target audience & their language
- Emotional tone (playful? serious? technical? warm?)
- Existing competitor names to differentiate from
- Constraints (length, pronounceability, cultural sensitivity)

**Step 2: Generate candidate names (50+ minimum)**
Use these naming strategies:
- **Portmanteau** — combining two words (Instagram = Instant + Telegram)
- **Metaphor** — abstract concept that captures the idea (Slack, Notion)
- **Invented** — new word that sounds right (Spotify, Figma, Vercel)
- **Real word repurposed** — existing word in new context (Apple, Notion, Linear)
- **Acronym/initialism** — only if it sounds good spoken (NASA yes, PCMCIA no)
- **Alliteration** — memorable repetition (PayPal, TikTok, Coca-Cola)
- **Compound** — two short words joined (Airbnb, YouTube, Snapchat)
- **Latin/Greek roots** — scholarly feel (Anthropic, Veritas, Codex)
- **Action verbs** — what it does (Zoom, Dash, Fetch)
- **Nature** — organic, trustworthy (Cedar, Aspen, Ember)

**Step 3: Filter candidates**
Score each name 1-10 on:
1. **Memorable** — can someone remember it after hearing once?
2. **Pronounceable** — no ambiguity in how to say it
3. **Spellable** — can someone type it from hearing it?
4. **Short** — ≤3 syllables ideal, ≤10 characters ideal
5. **Unique** — not confused with existing brands
6. **Domain available** — .com or exact-match TLD available
7. **No negative connotations** — check in major languages
8. **Trademark clear** — no obvious conflicts
9. **Scalable** — won't box you in if product expands
10. **Emotionally resonant** — does it FEEL right for the product?

**Step 4: Domain & handle availability check**
Run the domain checker for top 10 candidates:
```bash
./scripts/check-domain.sh "name1" "name2" "name3" "name4" "name5"
```

This checks: .com, .io, .app, .dev, .co, .ai TLDs + X/GitHub handles.

Also manually verify:
- **Trademark:** https://www.ic.gc.ca/app/opic-cipo/trdmrks/srch/home (Canada)
- **US Trademark:** https://tmsearch.uspto.gov/
- **Social handles:** X, Instagram, TikTok, YouTube, Reddit, GitHub
- **App stores:** Search Apple App Store + Google Play for conflicts
- **NPM/PyPI:** If it's a dev tool, check package registries

**Step 5: Present top 3-5 to stakeholders**
For each finalist, document:
- Name + pronunciation guide
- Why it works (emotional/strategic rationale)
- Domain availability (.com + alternatives)
- Social handle availability
- Potential logo directions (describe, don't design)
- Risk factors (trademark, cultural, competitor proximity)

Write findings to: `prompts/12-brand/naming-report.md`

**Step 6: Lock the name**
Once approved, update ALL references:
- `docs/brand/BRAND_GUIDE.md` — canonical name doc
- `prompts/00-shared-context/context.md` — broadcast to all agents
- `CLAUDE.md` — project name reference
- Escalate to PM to update all agent prompts with new name

## Phase 2: BRAND IDENTITY (After Name is Locked)

### Brand Guide (`docs/brand/BRAND_GUIDE.md`)
Create comprehensive brand documentation:

1. **Name** — official name, pronunciation, legal entity name
2. **Tagline** — one sentence, memorable (e.g., "One search bar for everything")
3. **Mission** — why this exists (1-2 sentences)
4. **Voice & Tone**
   - Personality traits (3-5 adjectives)
   - Writing examples (do this / not this)
   - Formality level (casual → formal scale)
   - Humor level (dry → playful scale)
5. **Audience** — who are they, what do they care about, what language do they use
6. **Positioning** — how we're different from competitors (one sentence)
7. **Color direction** — emotional intent (e.g., "calming teal, not aggressive red")
   - Suggest palette direction but defer to design/UI agent for exact values
8. **Typography direction** — personality match (e.g., "modern sans-serif, approachable")
9. **Naming conventions** — for features, products, tiers (e.g., Free/Pro/Enterprise)
10. **Content principles** — how to write copy, error messages, empty states
11. **Anti-patterns** — what the brand is NOT (e.g., "not corporate", "not gamified")

### Brand Voice Examples
Write 10+ real examples:
- App Store description (short + long)
- Error message tone
- Onboarding copy (first 3 screens)
- Marketing headline options
- Social media bio
- 404 page copy
- Email subject lines
- Changelog entry style
