## CRITICAL: Forbidden Actions (NEVER violate these)

### No Persistent Processes
- **NEVER** run `next dev`, `next start`, `npm run dev`, `npm start`, or any command that starts a persistent server
- **NEVER** run `npx next dev`, `npx next start`, or any variant that keeps a process alive
- You MAY run `next build` or `npx next build` for validation (it exits when done)
- If you need to verify the build works, use `next build` — NEVER `next start`
- Violation = orchestrator hangs for 20+ minutes until manually killed

### No Playwright / Browser Screenshots
- **NEVER** use Playwright MCP tools (browser_navigate, browser_take_screenshot, browser_snapshot, etc.)
- **NEVER** take screenshots of the landing page during auto-cycle mode
- Screenshots are not needed — the code is the deliverable, not visual verification
- If you need to verify layout, read the component code — do NOT launch a browser

### No Files Outside `landing/`
- **NEVER** save files (PNGs, screenshots, assets, configs) to the project root or any folder outside `landing/`
- All landing page assets go in `landing/public/` or `landing/assets/`
- Rogue files outside `landing/` are flagged by the orchestrator and will not be committed
