## Prompt Generation Workflow (CRITICAL — Follow Every Time)

Before writing ANY new prompt for 02, 03, or 04, you MUST complete these steps.
Writing prompts from memory alone produces stale instructions that waste agent time.

### Step 1: Read the Current Prompt
Read the agent's existing prompt file to understand:
- What tasks were assigned last time
- What "Current Status" section says
- What blockers were listed
- What "YOUR NEXT TASKS" section contains

### Step 2: Review What Actually Got Done
Compare the old prompt's task list against reality:
- `git log --oneline -20` — what commits landed since last prompt?
- `git status --short` — any uncommitted work?
- `gh issue list --milestone "M1.X" --state closed` — which issues got closed?
- Skim key files the agent was supposed to build/modify — do they exist? Are they substantial or stubs?

### Step 3: Identify Gaps & Drift
- Which assigned tasks were completed? Mark them DONE in the new prompt.
- Which tasks were partially done? Note what's left.
- Which tasks weren't touched? Why?
- Did the agent build anything NOT in the prompt?
- Are there new issues, PRs, or branch changes that shift priorities?

### Step 4: Write the New Prompt
- **Accurate "Current Status"** — reflects what actually exists in code and git
- **Completed work acknowledged** — so the agent doesn't redo finished work
- **Tasks ordered by priority** — unblocked work first
- **Specific file paths** — point to files that exist
- **Issue numbers** — every task tied to a GitHub issue
- **Dependencies called out**
- **Anti-drift guardrails**

### Step 5: Cross-Check Across Agents
- No two agents assigned overlapping file ownership
- Backend APIs that iOS depends on are actually built (or flagged as blocked)
- Design tokens match what 03 is using
- Issue numbers are real
