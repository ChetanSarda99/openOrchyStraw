**Objective:** Continuously audit the backend and iOS codebase for OWASP Top 10 vulnerabilities, data exposure, and security misconfigurations. Report findings with severity, file paths, and recommended fixes.

---

## Context

Rekall is an ADHD-first universal note aggregator. The backend handles sensitive user data: OAuth tokens for 10+ third-party services, personal notes, voice recordings, AI-processed content, and payment/subscription state.

**Tech stack:**
- **Backend:** Node.js + Express + TypeScript (strict), Prisma + PostgreSQL (pgvector), Redis + BullMQ
- **Auth:** Supabase Auth (JWT verification)
- **AI:** Claude 3.5 Sonnet, Voyage AI, AssemblyAI (all keys backend-only)
- **Storage:** AWS S3 (screenshots, audio)
- **Payments:** StoreKit 2 (App Store webhook + JWS verification)
- **Integrations:** Telegram, Notion, Instagram, Reddit, Twitter, Pocket, Google Calendar, SendGrid email
- **iOS:** Swift + SwiftUI (iOS 17+), Supabase Swift SDK

---

## Audit Scope

### OWASP Top 10
1. **Injection** — SQL injection via Prisma raw queries, command injection, prompt injection
2. **Broken Authentication** — JWT verification, session management, token storage
3. **Sensitive Data Exposure** — API keys in code, secrets in responses, PII leaks, token encryption
4. **XXE** — XML parsing in webhooks
5. **Broken Access Control** — userId scoping, admin routes, plan upgrade verification
6. **Security Misconfiguration** — CORS, headers, NODE_ENV defaults, error messages
7. **XSS** — Input sanitization, output encoding, S3 content-type validation
8. **Insecure Deserialization** — JSON parsing, webhook payloads
9. **Known Vulnerabilities** — npm audit, outdated packages
10. **Insufficient Logging** — Error tracking, audit trails

### Additional Checks
- SSRF in URL metadata extraction
- OAuth state parameter security
- AI prompt injection defenses
- File upload security (S3 content validation)
- Rate limiting adequacy (per-user, per-endpoint)
- Webhook signature verification (Telegram, SendGrid, App Store)
- Environment variable handling (defaults, required vs optional)
- Prisma query safety ($queryRawUnsafe usage)
- Per-user data isolation (cross-user access prevention)
