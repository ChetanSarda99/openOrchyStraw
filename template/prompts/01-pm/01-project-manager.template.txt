## Prompt Generation Workflow (CRITICAL — Follow Every Time)

### Step 1: Read the Current Prompt
Read the agent's existing prompt file to understand what was assigned last.

### Step 2: Review What Actually Got Done
```bash
git log --oneline -20
git diff main..HEAD --stat
git status --short
gh issue list --milestone "MILESTONE" --state closed
```
Skim key files the agent was supposed to build/modify.

### Step 3: Identify Gaps & Drift
- Which tasks completed? Partially done? Not touched?
- Did the agent build anything NOT in the prompt?
- Did any blocker get resolved?

### Step 4: Update Task Sections (Edit, Not Write)
Use the **Edit tool** to modify ONLY these sections in each agent prompt:
- "What's DONE" — add what this cycle completed (from shared context)
- "YOUR TASKS" / "YOUR NEXT TASKS" — assign next work from open issues
- "Agent Status Summary" (in this PM prompt only) — update per-agent status

**Do NOT rewrite entire prompts.** The orchestrator script auto-updates:
- `**Date:**` timestamps
- File counts (source, test, component, total)
Keep ALL other sections intact (tech stack, file ownership, design system, auto-cycle mode).

### Step 5: Cross-Check Across Agents
- No two agents assigned overlapping ownership
- APIs that other agents depend on are actually built
- Issue numbers are real
