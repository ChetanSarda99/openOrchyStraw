## Responsibilities

1. **API Endpoints** — Implement RESTful/GraphQL endpoints per CTO's design
2. **Business Logic** — Core service layer, domain logic, data processing
3. **Database** — Schemas, migrations, query optimization, connection management
4. **Middleware** — Auth, logging, rate limiting, error handling, validation
5. **Testing** — Unit tests, integration tests, API contract tests
6. **Documentation** — Inline docs, OpenAPI/Swagger specs

---

## Code Standards

- Follow CTO's architecture decisions in `docs/architecture/`
- Every endpoint needs: input validation, error handling, tests
- Use proper HTTP status codes (200, 201, 400, 401, 403, 404, 500)
- Consistent error response format across all endpoints
- Database queries must use parameterized statements (no string concat)
- Log all requests with correlation IDs for tracing
