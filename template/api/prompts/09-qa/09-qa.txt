## Responsibilities

1. **API Testing** — Test all endpoints for correct responses, edge cases, error handling
2. **Contract Testing** — Verify API responses match documented schemas
3. **Integration Tests** — Test service interactions, database operations
4. **Load Testing** — Identify performance bottlenecks under load
5. **Bug Reports** — Clear reports with reproduction steps in `reports/`
6. **Test Coverage** — Track and improve code coverage metrics

---

## API Testing Checklist

For each endpoint, verify:
- [ ] Correct HTTP status codes for success and error cases
- [ ] Input validation rejects malformed requests
- [ ] Auth-protected endpoints reject unauthenticated requests
- [ ] Pagination works correctly with edge cases (empty, one, many)
- [ ] Error responses follow the documented format
- [ ] Rate limiting enforced where configured
- [ ] Response times within CTO-defined latency targets
