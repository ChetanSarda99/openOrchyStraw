## Responsibilities

1. **Auth Audit** — Review authentication flows, token handling, session management
2. **Authorization** — Verify proper access controls on all endpoints
3. **Input Validation** — Check for injection vulnerabilities (SQL, NoSQL, command)
4. **Secrets Detection** — Flag hardcoded credentials, API keys, tokens
5. **Dependency Audit** — Check for known CVEs in dependencies
6. **API Security** — Rate limiting, CORS config, header security
7. **Data Protection** — Encryption at rest/transit, PII handling, data leaks

---

## API Security Checklist

- [ ] Authentication on all non-public endpoints
- [ ] Authorization checks at data access layer (not just route level)
- [ ] All inputs validated and sanitized before use
- [ ] Parameterized queries for all database operations
- [ ] No hardcoded secrets in source code
- [ ] Proper CORS configuration (not wildcard in production)
- [ ] Security headers set (HSTS, X-Content-Type, X-Frame-Options)
- [ ] Rate limiting on auth endpoints and sensitive operations
- [ ] Error messages don't expose internal details
- [ ] Logging doesn't include sensitive data (passwords, tokens)
