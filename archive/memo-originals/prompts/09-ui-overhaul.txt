## Design Philosophy

### Inspirations (from APP_BRANDING.md)
- **Linear.app** — Best-in-class SaaS design. Clean, purposeful animations. Monochrome + single accent color. Every pixel intentional.
- **Things 3** — The gold standard of iOS task apps. Calm hierarchy. Satisfying micro-interactions. Generous whitespace. Makes you WANT to use it.
- **Obsidian** — Content-first. Minimal chrome. Dark mode done beautifully. Information density without overwhelm.
- **Arc Browser** — Bold personality. Unique visual identity. Feels like nothing else.
- **Apple Health** — How to show complex data simply. Card layouts. Gentle animations.

### ADHD-First Design Principles
- **One clear action per screen** — What should the user do? Make it obvious.
- **Progressive disclosure** — Show only what's needed now. Details on demand.
- **Breathing room** — 16pt minimum padding. Generous spacing. Don't cram.
- **Visual hierarchy** — 3 levels max per screen (title, content, metadata)
- **Reduced cognitive load** — Fewer choices visible, more impact per choice
- **Calm > exciting** — This is a daily tool, not a game. It should feel like a warm library, not a casino.

### Color System (MemoTheme.swift — source of truth)
- Primary: #1A7A6D (deep teal) — trust, calm
- Accent: #E8734A (coral) — action, energy (CTAs only)
- Dark BG: #0D1B2A (deep navy, NOT pure black)
- Dark Surface: #1B2838 (card backgrounds)
- 8 area colors: Health red, Mind purple, Career blue, Money green, Relationships pink, Learning orange, Projects amber, Home brown

### How to Use Pattern Libraries (Mobbin, Screenlane, Refero)

When auditing a screen, ALWAYS include a "Reference Patterns" section:

```markdown
## Reference Patterns (from Mobbin/Screenlane)

### Search Screen
- Search: Mobbin → Filter: "Search" + "iOS" + "Dark mode" → study Linear, Things 3, Notion patterns
- Look at: search bar placement, empty state, result card density, filter chip patterns

### Onboarding
- Mobbin → Filter: "Onboarding" + "iOS" → study Calm, Headspace, Arc (ADHD-friendly, calm flows)
- Look at: step indicators, illustration style, copy density per screen, permission ask timing

### Settings
- Mobbin → Filter: "Settings" + "iOS" → study Apple Health, Arc, Bear (clean settings hierarchy)
```

For EVERY P0 screen audit, include:
1. **Mobbin search query** — exact filter terms to find relevant patterns
2. **3 reference apps** — specific apps to study for that screen type
3. **What to steal** — specific patterns/layouts from those references
4. **What to avoid** — patterns that don't fit ADHD-first design

### How to Use Framer

For interactive prototyping before code:
1. Build key flows in Framer (onboarding, search → detail, capture)
2. Test scroll feel, animation timing, transition smoothness
3. Export as video for stakeholder/tester review
4. Use Framer's "Smart Animate" to prototype spring animations matching MemoTheme values
5. Framer prototypes double as marketing assets (landing page demos, App Store preview video source)

### How to Use Lottie/Rive for iOS Animations

For animations that need to feel premium:
1. Search LottieFiles for: loading, success checkmark, confetti, empty state illustrations
2. Customize colors to match MemoTheme (teal primary, coral accent)
3. Export as .json, add to Xcode project, play via `LottieView` (lottie-ios package)
4. Rive for stateful animations: tab bar transitions, capture mode switching, onboarding illustrations
5. Both are lightweight and GPU-accelerated on iOS
